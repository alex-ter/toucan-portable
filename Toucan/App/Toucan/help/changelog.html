Changelog¶

3.1.8 (2020-05-01)¶

  • Fixed incorrect base file name in differential mode (bug #28)

  • Translations: updated JP translation (thanks maboroshin)

3.1.7 (2019-08-13)¶

  • Fixed recursive directory/file deletion in Sync and removed duplicated removal message (bug #22)

  • Updated bundled 7-Zip binary, it’s version 19.00 now (the latest available)

  • Help: corrected formatting in Secure description (thanks d-faure)

3.1.6 (2018-09-02)¶

  • Fixed an exception appearing when switching languages (bug #16)

  • Added an option for not logging the “Skipped” sync messages (forum request and bug #15)

  • Updated bundled 7-Zip binary, it’s version 18.05 now (the latest available)

  • Translations: updated DE and IT translations (thanks marwelix and eagleman)

3.1.5 (2017-06-17)¶

  • Toucan main executable now isn’t compressed and is dynamically linked to C/C++ runtime, libraries are supplied within the installer. This is to follow the PortableApps main team’s approach and avoid antivirus false-positives.

3.1.4 (2016-06-19)¶

  • Toucan is still alive! :)

  • documentation: fixed command line parameters description (bug #1)

  • documentation: added description of “anti-items” (bug #4)

  • scripts: fixed print command and exposed all message types available internally (bug #8)

  • backup: fixed job stdout polling logic to capture quick ones (bug #3)

  • backup: fixed ratio slider text not updating upon app start

  • translations: refreshed all PO files, updated RU and PL ones

  • translations: updated IT translation (eagleman)

3.1.3 (2014-01-19)¶

  • Fixed “–log” option not working when used with non-GUI mode (SF bug #7)

  • Minor documentation fixes (Toucan version strings mostly)

3.1.2 (2013-11-17)¶

  • Fixed wxWidgets assert being triggered when a whole drive was added to a backup selection (SF bug #6)

  • Fixed bug with UI looking flat like Win95 even on newer OSes with themes enabled (SF bug #4)

3.1.1 (2013-09-22)¶

  • Fixed bug with Backup in Complete or Update modes only including the first source file/dir into the resulting archive (SF bug #2)

  • Added description of Mirror mode for Backup into documenation (SF bug #1)

  • Fixed several error handling cases for Restore mode (errors were triggered when not needed and vice versa)
